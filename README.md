## 3. 분석 시나리오 및 데이터 수집

### 3.1 데이터 수집 도구 검토 및 한계점
초기 데이터 수집 단계에서 **다음(Daum) 뉴스**와 **네이버(Naver) 뉴스 API**를 검토하였으나, 다음과 같은 기술적 제약으로 인해 최종 수집 도구에서 제외하였습니다.

* **네이버 뉴스 API 검색의 한계**
    * `start` 파라미터 제약으로 인해 검색어당 **최대 1,000건**까지만 접근 가능하여 대규모 데이터 확보가 어려움.
* **네이버 뉴스 API 기간 검색 불가**
    * 특정 날짜 범위를 지정하는 파라미터가 부재하여, 시계열 트렌드 분석을 위한 정밀한 수집이 불가능함.
* **다음(Daum) 뉴스 수집의 효율성 저하**
    * 검색 결과가 각 언론사별 개별 사이트(Outlink)로 연결되어 있어, **상이한 HTML 구조**에 언론사별 개별 파싱이 필수적인 기술적 난점



### 3.2 Selenium 기반 데이터 수집 전략
API의 제약을 극복하고 규모를 확보하기 위해 **Selenium 기반의 웹 크롤러**를 설계하여 활용.

* **일자별 정밀 수집:** 네이버 뉴스 검색의 기간 설정 기능을 자동화하여, API 제한 없이 일자별 뉴스 헤드라인 및 본문 링크를 전수 수집함.
* **데이터 일관성 확보:** '네이버 뉴스(In-link)' 형식을 제공하는 기사만 수집하여, 텍스트 데이터의 구조적 일관성을 유지함.
* **시계열 분석 최적화:** AI 열풍 전후(2021년 vs 2025년 등)의 산업 구조 변화를 비교하기 위한 충분한 샘플 사이즈 확보가 가능함.
