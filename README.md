## 3. 기술적 한계 및 전략적 해결 방안

### 3.1 수집 도구의 한계점 (Technical Constraints)

* **API 제약**: 네이버/다음 API는 검색 결과가 제한적(최대 1,000건)이며, 과거 특정 시점의 데이터를 전수 조사하기에 부적합함.
* **비정형 구조**: 언론사별로 상이한 HTML 태그 구조로 인해 통합 파서(Parser) 구현 시 개별 파싱이 필수적이라 기술적 난점.

### 3.2 해결 전략: Selenium & BeautifulSoup 하이브리드 크롤링

* **일자별 쿼리 분할**: API의 제한을 피하기 위해 Selenium으로 날짜별 페이지를 자동 제어하여 데이터 전수 수집.
* **In-link 필터링**: 네이버 뉴스 플랫폼 내에서 제공되는 기사만 수집하여 데이터 포맷의 일관성을 확보하고 크롤링 속도를 개선.

### 3.3 분석의 한계 (Analytical Limitations)

* **채용 데이터의 수집 한계**: 해당 데이터는 API는 존재하나 채용사이트에서 요청 후 답변이 없어 수집이 불가능. 워크넷 API인 경우 일부 항목만 수집이 가능하여 제외
* **뉴스 데이터 한계**: 초기 데이터 수집 단계에서 **다음(Daum) 뉴스**와 **네이버(Naver) 뉴스 API**를 검토하였으나, 다음과 같은 기술적 제약으로 인해 최종 수집 도구에서 제외. 

* **네이버 뉴스 API 검색의 한계**
    * `start` 파라미터 제약으로 인해 검색어당 **최대 1,000건**까지만 접근 가능하여 대규모 데이터 확보가 어려움.
    * 특정 날짜 범위를 지정하는 파라미터가 부재하여, 시계열 트렌드 분석을 위한 수집이 불가능함.
* **다음(Daum) 뉴스 수집의 한계**
    * 검색 결과가 각 언론사별 개별 사이트(Outlink)로 연결되어 있어, **상이한 HTML 구조**에 언론사별 개별 파싱이 필수적인 기술적 난점

## 4. 주요 분석 결과 (Key Insights)

* **관심도의 변곡점**: 2020년 10월 학과 명칭 변경 시점과 코로나19 확산 시기가 맞물리며 디지털 전환(Digital Transformation) 관련 담론이 폭발적으로 증가.
* **도메인 확장**: 과거 '통계'가 행정 및 경제 지표 중심이었다면, 현재는 '플랫폼 서비스'와 '글로벌 금융 시장'으로 영역이 확장됨.
* **생태계 모델**: [데이터 수집 - 플랫폼 개발 - 서비스 제공]으로 이어지는 선순환 구조가 네트워크 분석 결과 뚜렷하게 관찰됨.
